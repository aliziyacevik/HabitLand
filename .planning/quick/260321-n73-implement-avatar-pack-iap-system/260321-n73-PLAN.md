- Do NOT add Codable conformance to AvatarType itself (associated values make this complex); use rawStorage for persistence.

**Create `HabitLand/Services/AvatarStoreManager.swift`:**

Follow ProManager pattern exactly:
- `@MainActor final class AvatarStoreManager: ObservableObject`
- `static let shared = AvatarStoreManager()`
- Product IDs: `static let animalsPackID = "com.azc.HabitLand.avatar.animals"`, `static let framesPackID = "com.azc.HabitLand.avatar.frames"`
- `@Published private(set) var products: [Product] = []`
- `@Published private(set) var animalsUnlocked: Bool` — read from UserDefaults key "avatarAnimalsUnlocked" in init
- `@Published private(set) var framesUnlocked: Bool` — read from UserDefaults key "avatarFramesUnlocked" in init
- `#if DEBUG @Published var debugAllUnlocked = false #endif` — in DEBUG and screenshotMode, return true for all unlock checks
- `func loadProducts() async` — load both product IDs via Product.products(for:)
- `func purchase(_ product: Product) async throws -> Bool` — same pattern as ProManager.purchase; on success, set corresponding UserDefaults flag and update published property
- `func isAvatarUnlocked(_ type: AvatarType, userLevel: Int) -> Bool`:
  - `.initial` -> always true
  - `.animal(.fox)` -> unlocked if userLevel >= 5 OR animalsUnlocked
  - `.animal(.owl)` -> unlocked if userLevel >= 10 OR animalsUnlocked
  - All other `.animal` -> unlocked if animalsUnlocked
  - `.frame(.crown)` -> unlocked if userLevel >= 15 OR framesUnlocked
  - All other `.frame` -> unlocked if framesUnlocked
  - In DEBUG+screenshotMode or debugAllUnlocked -> always true
- `func unlockReason(_ type: AvatarType) -> String?` — returns nil if unlocked, otherwise "Level 5", "$0.99", etc.
- `private func listenForTransactions() -> Task<Void, Error>` — mirror ProManager pattern, update unlock flags on transaction
- `private func updatePurchasedProducts() async` — check currentEntitlements for both pack IDs, update UserDefaults + published bools
- Call loadProducts and updatePurchasedProducts in init via Task, same as ProManager.
