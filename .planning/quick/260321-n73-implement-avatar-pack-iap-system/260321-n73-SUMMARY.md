## Decisions Made
- AvatarType uses rawStorage String encoding ("animal.fox", "frame.stars") instead of Codable, since SwiftData cannot store associated-value enums directly
- Avatar packs use non-consumable IAP with UserDefaults for unlock state persistence (mirrors ProManager pattern)
- Three avatars unlock by level progression (fox at 5, owl at 10, crown frame at 15) to reward free users
- SocialFeedView left using .initial since feed entries use a local struct without Friend model access
- InviteFriendsView and CreateChallengeView left unchanged (contacts/selection context where .initial default is correct)
